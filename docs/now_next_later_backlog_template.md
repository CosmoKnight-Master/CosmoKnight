# Now Next Later Backlog Template

## Purpose
Use this board to keep execution focused.

- `Now`: Active work only.
- `Next`: Ready and sequenced, not started.
- `Later`: Parked ideas and future scope.

## Operating Rules
- Maximum `Now` items: 3.
- Only one `Now` item can be marked `In Progress`.
- Any new request goes to `Later` first.
- To add a new `Now` item, close or move one current `Now` item.
- Every `Now` item must link to acceptance criteria.

## Weekly Cadence
- Monday: confirm `Now`, reorder `Next`, review `Later`.
- Mid-week: update status and risks.
- Friday: demo completed work, triage bugs, roll forward.

## Priority Scale
- `P0`: Blocker for current release.
- `P1`: High value in current release.
- `P2`: Important but deferrable.
- `P3`: Nice-to-have.
